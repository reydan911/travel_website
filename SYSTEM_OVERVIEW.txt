TRAVELBUDDY SYSTEM OVERVIEW
===========================

This project is a Spring Boot + Thymeleaf travel booking demo. It provides
user login, flight/hotel browsing, bookings, and AI-style recommendations.
All data is stored in MySQL and seeded from SQL files in src/main/resources.

MAIN FEATURES
-------------
1) Flights
   - Search and filter by origin, destination, date, and price.
   - Results are pulled from the flights table using JPA Specifications.

2) Hotels
   - Search and filter by location, stars, and price.
   - Results are pulled from the hotels table using JPA Specifications.

3) Destinations
   - Destination pages show hero images, highlights, activities, and maps.
   - Images and content come from destinations, destination_highlights,
     and destination_activities tables.

4) Bookings and Add-ons
   - Users can book flights or hotels and attach add-ons.
   - Booking data is stored in bookings and booking_add_ons tables.
   - Users can cancel bookings, but cannot remove only the flight or hotel.
   - Admin can remove cancelled bookings from the admin panel.

5) Recommendations (AI Suggestions page)
   - Recommendations are computed deterministically from database data:
     flights and hotels are scored by region, budget band, travel style,
     interests, and availability.
   - A destination anchor (city) is used so hotels and flights match.
   - Trip bundles pair flights and hotels from the same destination.

6) Preferences persistence
   - Preferences are saved when the AI Suggestions form is submitted.
   - Preferences are also updated automatically after flight or hotel bookings
     (preferred destination is set to the booked city).

DOES THIS USE AI?
----------------
Yes, but only in a limited way:
- The core matching logic is rule-based and deterministic (no external AI).
- A short natural-language summary can be generated by Groq (if configured).
- The AI summary is optional; if Groq is disabled or fails, the system falls
  back to a non-AI summary.

Where AI is used:
- RecommendationService.buildAiSummary() (GroqService)

Where AI is NOT used:
- Flight/hotel search
- Destination pages
- Booking flow
- Matching logic for recommendations
